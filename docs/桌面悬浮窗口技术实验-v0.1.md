# 代办：桌面悬浮窗口技术实验 v0.1

> 状态：可运行技术壳、任务优先界面精简与单屏真实桌面冒烟已通过；多显示器完整实机矩阵待补  
> 更新时间：2026-07-18  
> 技术路径：Tauri 2 + 原生 HTML/CSS/JavaScript  
> 报告边界：本报告只评价桌面窗口生命周期；共享桌面壳随后已承载独立的本地账本实验，仍不接 AI 或日历

## 1. 实验结论

Tauri 2 可以继续作为“代办”第一版桌面技术路径，当前不需要切到 Electron。已经跑通：

- 无边框透明悬浮窗与始终置顶；
- 冷启动和自动状态变化不抢键盘焦点；用户从托盘打开时恢复完整展开面板并允许输入；
- 展开、当前任务胶囊、仅宠物、贴边露出四种原生窗口尺寸；当前产品入口只开放展开与任务胶囊；
- 隐藏到托盘、从托盘恢复和明确退出的不同语义；
- Rust 窗口状态变化主动通知前端，托盘恢复后原生尺寸与页面模式保持一致；
- 位置保存、重启恢复和恢复到可用工作区的校正逻辑；
- 负坐标副屏、拔掉副屏和超大窗口的纯函数边界测试。

当前结论不是“正式桌面应用已经完成”。共享桌面壳现已接入第二个实验的创建、未完成标题修改、完成、删除、撤销和金币账本，但本报告不把这些能力计入窗口实验结论；100%、150%、200%、多显示器拔插和四种任务栏方向的人工实机矩阵仍未完成。

## 2. 为什么选择这条方案

### 目标

先回答一个风险最高的问题：极简原生方案能否作为安静、长期常驻、可恢复的 Windows 桌面工具存在。

### 选择 Tauri 2

Tauri 直接提供窗口、显示器、托盘和鼠标事件接口，并复用系统 WebView2，适合小型常驻工具。当前机器已经具备 Rust、MSVC、Windows SDK 和 WebView2，因此不需要额外安装系统工具链。

窗口实验当时只新增一个 npm 开发依赖 `@tauri-apps/cli`，Rust 侧使用 `tauri` 和官方 `tauri-plugin-window-state`，没有引入 React、Vite、数据库或大型应用框架。共享包后来为第二个实验增加 `rusqlite`，其选择和边界不属于本报告，见[《本地事件历史、金币账本与异常恢复技术实验 v0.1》](./本地事件历史、金币账本与异常恢复技术实验-v0.1.md)。

### 依赖边界

窗口子链路已经按职责拆分：

```text
desktop/index.html + styles.css
              │
              ▼
desktop/app.js（前端组合入口）
              ├─ state / selectors / views
              ├─ ledger-contract / ledger-session
              ├─ shell-controller / window-controller
              ├─ pet-component（保留技术模式，主界面暂不使用）
              └─ infrastructure/tauri-gateway / outbox-store
                              │ 只发送窗口或账本意图
                              ▼
src-tauri/src/app.rs（应用组合根）
              ├─ desktop/commands.rs（Tauri 薄命令）
              ├─ desktop/state.rs（窗口状态与规格）
              ├─ desktop/window.rs（窗口 / 显示器适配）
              ├─ desktop/tray.rs（托盘适配）
              ├─ runtime_profile.rs（正常 / 冒烟运行配置）
              ├─ frontend_probe.rs（真实前端就绪 IPC）
              └─ integration_smoke.rs（仅联合冒烟编排）

src-tauri/src/window_geometry.rs
              └─ 不依赖 Tauri 的位置选择、贴边和工作区校正
```

任务、事件和金币领域模型现已落在独立 `ledger` 模块，只依赖项目自己的接口，不直接依赖 Tauri；宠物领域暂停。若未来更换桌面壳，核心业务不需要一起迁移。

Tauri 命令统一在组合根中使用模块限定路径注册，例如 `crate::desktop::commands::set_window_mode` 与 `crate::frontend_probe::report_frontend_ready`。这不是命名偏好：命令宏生成的配套符号仍属于命令的定义模块，跨模块注册时若只导入函数并使用短名称，`generate_handler!` 会在错误的模块路径下解析配套符号。后续新增命令继续保持“定义模块内声明、`app.rs` 用模块限定路径注册”，不在 `main.rs` 堆业务入口。

Electron 只保留为显式备选：若后续实机证明 Tauri 无法稳定做到常驻状态不抢焦点，再单独做 Electron 对照，不设置静默回退。

## 3. 状态与流程

### 状态

| 状态 | 原生窗口尺寸（逻辑像素） | 焦点策略 | 当前产品入口 |
|---|---:|---|---|
| 展开 | 420 × 520 | 仅用户主动展开时可聚焦 | 胶囊或托盘展开 |
| 当前任务胶囊 | 310 × 92 | 不可聚焦 | 默认常驻或收起 |
| 仅宠物 | 104 × 104 | 不可聚焦 | 不开放，仅保留技术模式 |
| 贴边露出 | 72 × 104 | 不可聚焦 | 不开放，仅保留技术模式 |
| 隐藏到托盘 | 无桌面窗口 | 不适用 | 胶囊右上角窗口级向下箭头、更多菜单或托盘 |
| 退出 | 窗口、托盘、进程均结束 | 不适用 | 托盘明确退出 |

### 第 1 轮：冷启动

1. 官方窗口状态插件只恢复位置，不恢复尺寸和可见状态。
2. 应用固定以当前任务胶囊启动。
3. 保存位置仍在现有显示器时继续使用；否则选择最近工作区并校正。
4. 首次启动没有保存位置时，放到主显示器工作区右下角，保留 12 物理像素边距。
5. 窗口设置为不可聚焦后再显示。

输出：右下角胶囊可见、始终置顶、任务栏不新增普通窗口入口、原应用继续接收键盘输入。

### 第 2 轮：用户主动展开

1. 用户点击胶囊，或从托盘左键 / 菜单“打开代办”进入完整展开面板。当前主界面不提供宠物展开入口。
2. 窗口从 310 × 92 变为 420 × 520。
3. 根据离哪一条边更近，保持对应边缘锚点并向屏幕内部展开。
4. 只有这一条主动路径允许窗口变为可聚焦。

输出：完整面板位于当前工作区内，不越过任务栏。

### 第 3 轮：被动常驻

1. 收起、仅宠物或贴边时，将窗口设回不可聚焦。
2. 贴边态缩小真实原生窗口边界，而不是保留一个大透明窗口。
3. 当前被动常驻只显示任务胶囊；未来动画或奖励反馈也只能改变页面内容，不能改变焦点策略。

输出：桌面常驻但不挡住大块鼠标区域，也不夺走键盘输入。

### 第 4 轮：隐藏、恢复与退出

1. “隐藏到托盘”隐藏桌面窗口，进程与托盘保留。
2. 托盘左键或菜单“打开代办”统一调用展开恢复路径，窗口变为完整展开态并允许输入。
3. Rust 完成窗口模式切换后发送 `window-status-changed`，前端订阅该事件并同步页面模式，避免原生窗口已经展开但页面仍停留在胶囊布局。
4. “退出代办”先保存位置，再结束窗口、托盘和进程。

## 4. 边界与不变式

- 收起、隐藏、退出始终是三个不同动作。
- 冷启动固定进入不可聚焦胶囊；托盘左键与“打开代办”固定进入可聚焦展开态。
- 除点击胶囊或从托盘主动打开外，窗口不得成为前台键盘目标。
- 冷启动、状态切换和托盘恢复后，可见态必须完整位于某个现存显示器工作区内。
- 自由拖动期间不逐次钳制窗口，避免妨碍跨屏移动；若用户主动拖到屏外，可从托盘恢复。拖动结束自动吸附或回收仍是待验证增强项。
- 展开时保持最近边缘锚点并向屏幕内部生长。
- 位置使用物理像素保存；界面尺寸使用逻辑像素，再按当前缩放换算。
- 原生窗口模式与前端页面模式必须通过 `window-status-changed` 保持一致。
- 旧实现不会作为新窗口逻辑的静默回退。

## 6. 运行方式

开发运行：

```powershell
npm.cmd run desktop:dev
```

自动冒烟会短暂显示窗口、完成检查并自动退出：

```powershell
npm.cmd run desktop:smoke
```

本轮发布版二进制仓库相对路径：

```text
src-tauri/target/release/zuoban-desktop-spike.exe
状态：版本 0.1.0，用户可见名称与 Windows 文件元数据均为“代办”，托盘使用无底板透明图标；待办与完成记录搜索、胶囊布局、窗口隐藏、列表内部滚动、标题与可选期限修改和 schema v4 均已通过自动门禁与合成 smoke。
发布口径：本机构建大小、进程和历史哈希不进入源码文档；GitHub Release 如提供二进制，应单独附对应公开产物的校验值。
```

Windows 安全软件可能误报未签名的本地调试 EXE，并让 `cargo run` 返回“拒绝访问”。遇到此类问题时先核对进程与 `src-tauri/target` 构建目录；如确需排除，仅对可信开发目录设置最小范围规则，不要长期关闭安全软件。
