# 代办：视觉方向决策 v0.3

> 决策日期：2026-07-13  
> 状态：已确认

## 决策

三种视觉设计全部保留，极简原生风优先完成并作为第一版实现与验收基线。

| 方案 | 定位 | 优先级 |
|---|---|---:|
| 极简原生风 | 当前主方案 | P0 |
| 深色透明风 | 后续候选皮肤 | Backlog |
| 像素桌宠风 | 后续候选皮肤 | Backlog |

## 原因

极简原生风更适合长期常驻桌面：任务信息清楚、收起后干扰较小，也更容易先验证“不抢焦点、快速记录、窗口位置恢复”等桌面工具的核心风险。宠物保留鲜活颜色和有限动画，避免界面因为过度克制而失去陪伴感。

## 保留策略

- 不删除深色透明和像素桌宠设计稿。
- 第一版不并行实现三套完整主题。
- 极简原生先确定组件结构、状态语义、键盘行为和可访问性。
- 后续皮肤复用同一套业务组件和状态模型。
- 皮肤只能改变视觉资源和有限动效，不得复制任务、奖励和历史逻辑。
- 暖色手账风作为更早的历史探索稿单独存档，不属于本次确认保留的三套设计。

## 第一版验收边界

第一版只以极简原生风验收。深色透明和像素桌宠是否适配，不影响第一版发布。

极简原生的四个发布阻断项：

1. 悬浮窗不能抢走其他应用的键盘焦点。
2. 收起态不能遮挡系统托盘和常用桌面操作。
3. 快速记录应在约三秒内完成。
4. 多显示器、系统缩放和重启后的窗口位置必须可靠恢复。

完整交互与状态矩阵见[《界面与交互设计说明 v0.2》](./界面与交互设计说明-v0.2.md)。
