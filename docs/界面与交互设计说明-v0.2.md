## 1. 视觉决策与保留策略

三种视觉设计全部保留：

| 方案 | 当前角色 | 当前动作 |
|---|---|---|
| 极简原生风 | 主方案、第一版实现与验收基线 | 优先完成待办主流程与日常试用手感 |
| 深色透明风 | 候选皮肤 | 保留视觉稿，暂不平行补全 |
| 像素桌宠风 | 候选皮肤 | 保留视觉稿，暂不平行补全 |

第一版只实现一套组件结构和业务逻辑。后续皮肤共享任务状态、页面结构、按钮语义、键盘行为和可访问性规则，只允许替换颜色、字体、圆角、材质、图标、宠物资源和有限动效。

第一轮暖色手账风继续作为历史探索稿保存，但不属于本次确认保留的三套方案，也不再作为默认入口。

## 2. 极简原生风设计目标

主界面像一件长期驻留桌面的系统工具：安静、清楚、可预期。当前任务面板使用中性层次，暖色只作为轻量品牌点；宠物不占据待办主界面。

核心目标：

- 收起后不挡桌面：有待办时完成圆圈位于标题最前，标题可截断，按需出现的期限固定在同一行末尾；无待办时只显示“暂无待办”，不保留完成圆圈或期限。
- 展开后使用一份统一列表，不再拆分“现在做”和“接下来”。
- 每条待办都可直接勾选完成，并可用拖动或键盘调整顺序。
- 未完成待办可在原行修改标题和可选期限，不增加弹窗或常驻编辑按钮；无期限时不留日期占位。
- 每条待办都有低存在感的“×”删除入口；删除保留历史且不发金币。
- 记录一件事情只需要快捷键、输入、回车。
- 完成反馈短暂出现，不弹出结算窗口。
- 稍后再做、被卡住、不再做均有清楚去向，不使用责备文案。
- 后续恢复宠物能力时，它应有陪伴感，但不持续跳动、闪烁或播放声音。
- 历史和周报保持工具属性，不依赖 AI。

## 3. 产品界面分层

### 桌面常驻层

当前负责显示列表第一条待办、勾选完成和展开；完成圆圈位于任务标题最前，已有期限固定在标题行末尾，无待办时显示“暂无待办”并隐藏完成圆圈和期限。胶囊右上角的窗口级向下箭头可直接隐藏到托盘，不要求先展开，也不与任务删除操作混在一起。后续宠物能力不得增加这层的信息密度。

### 任务展开层

当前负责快速新增、统一待办列表、未完成标题修改、完成、重排、删除和完成记录入口。“暂时做不了”属于下一轮任务状态能力，尚未实现。

### 完整主窗口

这是后续独立窗口，计划负责全部任务、历史周报、迁移、备份和设置；宠物小屋暂停。它不是始终置顶窗口。

### 系统托盘

负责重新显示、临时隐藏和退出。托盘左键与菜单“打开代办”都恢复完整展开面板；系统托盘隐藏不等于退出程序。

## 4. 桌面生命周期

| 状态 | 可见内容 | 当前入口 | 说明 |
|---|---|---|---|
| 展开态 | 快速记录、统一待办列表、完成记录入口 | 点击胶囊、托盘左键或“打开代办” | 托盘恢复固定进入此状态 |
| 当前任务胶囊 | 完成圆圈、列表第一条待办、可选期限按钮；空列表显示“暂无待办” | 默认常驻或收起 | 完成圆圈位于标题最前；标题可截断；已有期限固定在同一行末尾并保留至少 24px 点击高度；空列表隐藏圆圈和期限 |
| 隐藏到托盘 | 桌面无窗口，托盘保留 | 胶囊右上角窗口级向下箭头、更多菜单或托盘 | 当前可用，无需先进入任务页面；与任务删除操作分离 |
| 仅宠物态 | 宠物与轻量状态点 | 主界面不开放 | 保留的技术模式，宠物阶段暂停 |
| 贴边露出态 | 窄标签或宠物的一部分 | 主界面不开放 | 保留的技术模式，暂不作为产品入口 |
| 退出 | 程序和托盘均结束 | 托盘菜单明确选择退出 | 当前可用 |

必须统一使用以下语言：

- **收起**：桌面仍能看到当前任务胶囊。
- **隐藏**：桌面不可见，只留系统托盘。
- **退出**：结束整个应用。

收起态建议宽度不超过 320 像素，默认位于任务栏上方并向屏幕内部展开。

## 5. 核心场景与数据边界

### 5.1 桌面执行

展开态只展示快速记录、统一待办列表和完成记录入口。每行由勾选框、可聚焦的任务标题、按需出现的紧凑期限标签、低存在感的“×”和右侧拖动柄组成；无期限任务不显示日期或占位，第一行只用轻量强调表示它也是胶囊任务，不使用独立的“当前任务”卡片。长列表只在待办列表内部滚动，录入区和底部完成记录入口保持固定；7px 滚动槽位于面板 16px 内边距之内，不接触窗口圆角。今日进度、金币、宠物和技术状态不占用主界面。

完成流程：

```text
勾选任意一条待办
→ 写入完成事件
→ 金币账本在后端同事务记账
→ 界面短暂显示“已完成”
→ 该项从列表移除；如果完成的是第一项，新的第一项进入胶囊
→ 在完成记录页以行末 `↶` 图标提供撤销，保留“撤销完成：任务名”的无障碍名称
```

撤销完成时同时回退奖励，不能重复发币。

调整顺序时拖动任务右侧手柄；纯键盘操作可聚焦手柄后按 `Alt+↑/↓`。重排按完整旧顺序和新顺序一次提交，失败时恢复原顺序；`reorder_tasks` 与其他写操作共用可靠操作箱，避免结果未知时重复改序。

待办较多时按 `Ctrl+F`，或从更多菜单选择“搜索待办”；快速记录区临时替换为共享搜索栏，仅按当前待办标题做不区分英文大小写的包含筛选，并用克制的暖色标出命中片段。完成记录页使用同一入口和位置，菜单文案切换为“搜索完成记录”，只筛选有效完成事件的标题快照。空查询保留搜索态并显示完整列表；无命中时分别显示“没有找到相关待办”或“没有找到相关完成记录”。`Esc` 或“取消”退出搜索，中文输入法组合期间不会用半成品刷新列表，也不会把候选框的 `Esc` 误当成退出。

搜索只改变当前页面的只读投影，不改变账本快照和原顺序。筛选结果仍可完成、修改、删除或撤销；排序柄在搜索态隐藏，拖动和 `Alt+↑/↓` 也由控制器明确拒绝，避免把局部结果提交成完整队列。完成、删除或改名让最后一个命中项消失时，焦点回到搜索框；“当前待办”标记仍严格以真实 `currentTask.id` 为准，不把筛选后的第一项冒充当前项。

删除流程：

```text
点击待办行的“×”
→ 任务从当前队列移除
→ 后端写入软删除事件并保留历史
→ 不写入金币奖励
```

当前删除入口保持轻量，不弹出金币或完成反馈，也不伪装成完成。前端暂未提供删除撤销入口。

标题修改流程：

```text
双击未完成待办标题，或聚焦标题后按 Enter / F2
→ 原位置切换为单行标题输入框并聚焦，同时显示可选日期控件
→ Enter 或输入框失焦时提交有效变化
→ 后端追加 title_updated 事件并刷新真实快照
→ 任务仍留在原顺序，不改变状态和金币
```

编辑时按 `Esc` 会恢复原标题并结束编辑，不提交命令，也不会继续触发窗口收起。空白标题不会提交：按 `Enter` 时保留编辑框并标记无效，失焦时取消；与原题相同的标题直接取消。输入法组合输入期间的 `Enter` 不得提前保存。标题修改仅面向当前统一列表中的立即待办，完成记录和其他非立即状态不提供入口。

期限展示与编辑流程：

```text
新任务默认 deadlineOn=null
→ 常态列表和胶囊不显示日期
→ 双击标题、按 Enter/F2，或点击任务行已有期限标签进入原位编辑
→ 点击胶囊标题行末尾的独立 edit-current-deadline 按钮时，一次完成展开面板并聚焦当前任务日期输入
→ 编辑期间隐藏原期限标签
→ 日期控件只接受 YYYY-MM-DD；清空表示 null
→ 有效变化通过 update_task_deadline 提交
→ 后端追加 deadline_updated，并刷新真实快照
→ 日期选择或清除成功后，焦点回到任务标题
```

已有期限按当前本地日期紧凑显示：“今天”“明天”、同年其他日期 `M/D`、跨年日期 `YYYY/M/D`、过去日期“逾期 N 天”。这些标签只做展示和编辑入口；设置、改期、清除、逾期或午夜刷新都不会自动排序、隐藏任务、提醒、通知、同步日历、发放或扣除金币。逾期和午夜刷新由前端从 `deadlineOn` 派生，不追加领域事件。期限与“稍后再做”的 `deferUntil` 完全独立，不决定任务是否留在立即执行队列。仅当前可见的立即待办可修改期限；完成、撤销完成和删除只改变原有状态语义，保留任务已有期限字段。

### 5.2 快速记录

当前入口是展开面板中的单行输入：输入一句话，按“记下”或回车后进入队尾。当前没有“全局新增待办”快捷键；`Ctrl+F` 已专用于当前待办页或完成记录页搜索。

标题是唯一必填项。快速记录中的空内容、重复回车和保存失败必须有明确但不打断的处理。行内标题编辑激活时，`Esc` 优先取消编辑；只有没有编辑态时，界面壳才继续处理收起临时页面或窗口的后续 Esc 语义。

### 5.3 后续：暂时做不了

以下状态语义已经设计，但当前桌面版本尚未开放命令和界面：

| 操作 | 第一版输入 | 处理结果 |
|---|---|---|
| 稍后再做 | 明天、下周、自定义时间 | 到期前跳过，不强制替换当前任务 |
| 被卡住 | 简短原因 | 进入阻塞区，可恢复处理 |
| 不再做 | 调整原因 | 进入已放弃，可重新打开 |

三种处理均不发金币、不扣金币，也不影响宠物成长；确认后下一条可执行任务成为当前任务。

### 5.4 暂停：宠物小屋

宠物阶段恢复后，第一版只包含金币、少量食物、成长值和成长阶段：

```text
完成任务获得金币
→ 花费金币喂食
→ 成长值增加
→ 达到阈值后成长
```

宠物不会死亡、掉级、饿坏或因长时间未使用责备用户。金币不足时只给温和提示。

### 5.5 完成记录、复制本周完成与后续周报

当前已实现独立完成记录页，展示完成标题、时间和撤销入口。每行标题可截断，完成时间固定在内容区末端，最右侧为 28 × 28px 的 `↶` 撤销图标；完成记录标题固定，仅记录列表内部滚动，并与待办页复用同一条内缩细滚动条规则。按 `Ctrl+F` 或更多菜单可把固定标题临时换成搜索栏，实时筛选有效完成记录；右侧总数仍表示全部有效完成记录，不被筛选结果改写。它读取本地事件账本，不依赖网络。

源码候选版本 `0.1.1` 在完成记录标题右侧增加低存在感文字按钮“复制本周”。用户点击后，应用按本地时区查询 `[本周一 00:00, 下周一 00:00)` 的有效完成事件，已撤销完成不会进入结果；输出固定为 `## 本周完成` 和 Markdown 编号列表。写入成功提示“已复制本周完成（N 项）”；本周没有有效完成记录时提示“本周还没有完成记录”，并且不覆盖现有剪贴板内容。

这只是完整非 AI 周报的第一个最小切片。后续周报再按“本周完成、进行中、阻塞事项、计划调整”分组，保留时间和原因，并继续支持：

- 今天、上周和自定义日期。
- 选择是否纳入单项。
- 查看单任务事件时间线。
- 复制纯文本或包含更多分组的 Markdown。
- 不依赖 AI 和网络生成基础周报。

### 5.6 本地数据边界

当前数据库结构为 v4。已有 v1、v2、v3 数据会按级迁移到 v4；磁盘旧库在迁移前通过 `VACUUM INTO` 生成并验证一次一致性备份，再执行结构迁移。这个备份只服务于数据库升级，不等于通用自动备份；手动导出、定期备份和未来更多版本迁移仍是后续能力。

## 6. 极简视觉系统

### 色彩

- 页面和系统背景：中性浅灰。
- 任务面板：白色或轻微系统材质。
- 文字：深灰，不使用纯黑大面积压迫。
- 交互强调：低饱和系统蓝或绿色。
- 当前暖色品牌点：只做小面积提示；未来宠物与金币启用后也不能扩散成大面积装饰。
- 错误与逾期：仅使用小面积提示，不使用大面积红色。

### 字体与密度

- 中文正文优先使用系统清晰字体。
- 数字使用清晰、稳定的字形，便于阅读数量和时间。
- 任务标题字号明显高于元信息，但不使用展示型大标题。
- 桌面层保持紧凑，完整主窗口保持舒适行高。

### 圆角与阴影

- 圆角克制，避免所有组件都变成胶囊。
- 阴影只表达悬浮层级，不能形成厚重卡片堆叠。
- 透明是增强效果，关闭系统透明后仍需清楚可用。

### 动效

- 当前完成反馈保持短暂且不打断操作。
- 后续喂食反馈建议为 0.6～1.2 秒，待机宠物只做小幅呼吸或眨眼。
- 支持减少动态效果。
- 默认无声音、无全屏撒花、无必须关闭的奖励弹窗。

## 7. P0 设计原型覆盖范围

以下流程必须在主方案原型中能够从入口走到结果：

- [x] 右下角收起态与任务展开态。
- [x] 快速记录、保存反馈和队尾语义。
- [x] 完成、金币变化、宠物反馈和下一任务。
- [x] 稍后再做、被卡住、不再做三种处理入口、原因与恢复结果。
- [x] 宠物小屋、喂食和成长进度。
- [x] 本周历史、完成撤销、调整记录和可查看的周报草稿。
- [x] 展开、胶囊、仅宠物、贴边和托盘隐藏的状态语义。

上述流程均已由设计原型覆盖，不等于当前生产实现。当前 Tauri 源码已经实现创建、待办与完成记录标题搜索、仅立即待办的行内标题修改、待办可选期限、任意待办完成、统一列表重排、软删除、撤销、本地完成记录、固定自然周的有效完成项 Markdown 复制和金币事务；搜索支持 `Ctrl+F` / 更多菜单、输入法组合态保护、安全高亮、空结果、焦点恢复和搜索态重排双重锁定。标题修改支持双击及 `Enter` / `F2` 进入、`Enter` 或失焦保存、`Esc` 取消和输入法组合态保护。日期控件只在编辑态出现，任务行期限标签可点击编辑；胶囊期限固定在标题行末尾，独立 `edit-current-deadline` 按钮会一次展开并聚焦当前任务日期输入，编辑期间隐藏旧标签，日期选择或清除成功后焦点回到任务标题。托盘左键与“打开代办”统一恢复展开态，窗口状态变化由 Rust 主动通知前端同步。仅宠物、贴边仍保留为技术模式但不从主界面开放。延期、阻塞、放弃、完整周报和宠物能力仍未实现或处于暂停状态；日历仍暂停，本地期限不是日历联动。

任务状态、事件历史和金币事务的冻结规则见[《任务状态与事件账本 v0.1》](./任务状态与事件账本-v0.1.md)。

## 8. P1 规则与开发验证

以下内容需要有明确规则，但不要求本轮全部做成高保真页面：

- 超长标题、任务很多和空队列。
- 标题修改的双击、键盘进入、失焦保存、取消、输入法组合态和重复提交保护。
- 期限控件的设置、改期、清除、同值拒绝，以及今天/明天/同年/跨年/逾期和午夜刷新展示。
- 今天、上周、自定义区间和跨月/跨年的完整周报。
- 多行迁移预览、重复标题和迁移结果。
- 备份恢复预览、成功和失败。
- 浅色、深色和高对比主题。
- 100%～200% 缩放、多显示器、任务栏不同方向；当前仅单屏 125% 实机通过。
- 拔掉副屏后的窗口回收；当前纯函数边界测试通过，实机待补。
- 全屏自动隐藏；单屏位置恢复已通过，整窗鼠标穿透已从产品与代码中移除。
- 减少动态效果和更完整的纯键盘操作；当前重排已支持 `Alt+↑/↓`。

## 9. 第一版发布阻断项

以下四项任一失败，极简原生版本都不适合作为长期桌面常驻工具：

1. 悬浮窗抢走其他应用的键盘焦点。
2. 收起态过大、遮挡系统托盘或常用按钮。
3. 快速记录不能在约三秒内完成。
4. 多显示器、缩放或重启后窗口位置无法可靠恢复。

## 10. 后续皮肤适配

极简原生主流程和状态稳定后，深色透明与像素桌宠按照同一状态矩阵适配。皮肤适配不能复制业务逻辑，也不能改变“完成”“隐藏”“阻塞”等操作语义。
