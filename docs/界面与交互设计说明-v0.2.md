## 1. 视觉决策与保留策略

三种视觉设计全部保留：

| 方案 | 当前角色 | 当前动作 |
|---|---|---|
| 极简原生风 | 主方案、第一版实现与验收基线 | 优先完成待办主流程与日常试用手感 |
| 深色透明风 | 候选皮肤 | 保留视觉稿，暂不平行补全 |
| 像素桌宠风 | 候选皮肤 | 保留视觉稿，暂不平行补全 |

第一版只实现一套组件结构和业务逻辑。后续皮肤共享任务状态、页面结构、按钮语义、键盘行为和可访问性规则，只允许替换颜色、字体、圆角、材质、图标、宠物资源和有限动效。

第一轮暖色手账风继续作为历史探索稿保存，但不属于本次确认保留的三套方案，也不再作为默认入口。

## 2. 极简原生风设计目标

主界面像一件长期驻留桌面的系统工具：安静、清楚、可预期。当前任务面板使用中性层次，暖色只作为轻量品牌点；宠物不占据待办主界面。软件图标使用“悬浮任务牌”：深色软件底座中叠放两张任务卡，珊瑚橙前景牌代表当前任务，不再以宠物或通用完成勾作为主识别符号。

核心目标：

- 收起后不挡桌面：有待办时完成圆圈位于标题最前，标题可截断，按需出现的期限固定在同一行末尾；无待办时只显示“暂无待办”，不保留完成圆圈或期限。
- 展开后使用一份统一列表，不再拆分“现在做”和“接下来”。
- 每条待办都可直接勾选完成，并可用拖动或键盘调整顺序。
- 未完成待办可在原行修改标题和可选期限，不增加弹窗或常驻编辑按钮；无期限时不留日期占位。
- 每条待办都有低存在感的“×”删除入口；删除保留历史且不发金币。
- 一级子代办使用渐进披露：无子项时不显示进度，但父行固定保留唯一的低存在感 `＋`；有子项时紧凑进度只负责展开或收起，具体操作留在一级缩进列表。
- 子代办使用单次添加态，并支持原位修改、软删除、完成、撤销和同组重排；父项仍是期限、最终完成和金币奖励的主体。
- 记录一件事情只需要快捷键、输入、回车。
- 完成反馈短暂出现，不弹出结算窗口。
- 稍后再做、被卡住、不再做均有清楚去向，不使用责备文案。
- 后续恢复宠物能力时，它应有陪伴感，但不持续跳动、闪烁或播放声音。
- 历史和周报保持工具属性，不依赖 AI。

## 3. 产品界面分层

### 桌面常驻层

当前负责显示列表第一条待办、勾选完成和展开；完成圆圈位于任务标题最前，已有期限固定在标题行末尾，无待办时显示“暂无待办”并隐藏完成圆圈和期限。胶囊右上角的窗口级向下箭头可直接隐藏到托盘，不要求先展开，也不与任务删除操作混在一起。后续宠物能力不得增加这层的信息密度。

### 任务展开层

当前负责快速新增、统一待办列表、一级子代办的渐进展开与组内操作、未完成标题修改、完成、重排、删除和已完成入口。“暂时做不了”属于下一轮任务状态能力，尚未实现。基础子代办能力已随 `v0.1.2` 发布，固定新增入口与空新增取消折叠已随 `v0.1.3` 发布；标题栏、操作区、底部入口和软件图标的收口已进入 `v0.1.4` 发布候选。

### 完整主窗口

这是后续独立窗口，计划负责全部任务、历史周报、迁移、备份和设置；宠物小屋暂停。它不是始终置顶窗口。

### 系统托盘

负责重新显示、临时隐藏和退出。托盘左键与菜单“打开待办”都恢复完整展开面板；系统托盘隐藏不等于退出程序。

## 4. 桌面生命周期

| 状态 | 可见内容 | 当前入口 | 说明 |
|---|---|---|---|
| 展开态 | 快速记录、统一待办列表、已完成入口 | 点击胶囊、托盘左键或“打开待办” | 托盘恢复固定进入此状态 |
| 当前任务胶囊 | 当前父项或“父项 / 第一条未完成子代办”、整组进度、父项可选期限；空列表显示“暂无待办” | 默认常驻或收起 | 圆圈完成胶囊正在显示的动作；子项全部完成后重新指向父项；标题可截断，期限仍属于父项；空列表隐藏圆圈、进度和期限 |
| 隐藏到托盘 | 桌面无窗口，托盘保留 | 胶囊右上角窗口级向下箭头、更多菜单或托盘 | 当前可用，无需先进入任务页面；与任务删除操作分离 |
| 仅宠物态 | 宠物与轻量状态点 | 主界面不开放 | 保留的技术模式，宠物阶段暂停 |
| 贴边露出态 | 窄标签或宠物的一部分 | 主界面不开放 | 保留的技术模式，暂不作为产品入口 |
| 退出 | 程序和托盘均结束 | 托盘菜单明确选择退出 | 当前可用 |

必须统一使用以下语言：

- **收起**：桌面仍能看到当前任务胶囊。
- **隐藏**：桌面不可见，只留系统托盘。
- **退出**：结束整个应用。

收起态建议宽度不超过 320 像素，默认位于任务栏上方并向屏幕内部展开。

## 5. 核心场景与数据边界

### 5.1 桌面执行

展开态只展示快速记录、统一待办列表和已完成入口。父行由勾选框、可聚焦标题、按需出现的子项进度与期限、固定的低存在感 `＋`、删除入口和右侧拖动柄组成；没有子代办时不显示 `0/0` 或展开箭头，但仍可直接从固定 `＋` 新增。存在子代办时，点击进度展开或收起一级缩进列表，`＋` 仍只负责新增；子列表底部不重复放置文字入口。展开只在固定高度的父行下方增减子列表，不改变父行控件的纵向位置。当前待办左侧橙色标记始终相对父行首行居中，编辑态增高时随首行移动，展开子列表时不延伸到子项区域。父行与子行都把可选按钮收进靠右自适应操作区；按钮隐藏后退出布局，剩余按钮仍靠右，空余宽度归还标题。子列表与父列表共用同一个内部滚动区，不做卡片套卡片。无期限任务不显示日期或占位，第一行只用轻量强调表示它也是胶囊任务，不使用独立的“当前任务”卡片。录入区和底部已完成入口保持固定；“已完成”、总数和箭头组成右侧紧凑按钮，三者都可点击进入，整行其余空白不响应点击，避免操作列表底部时误触。7px 滚动槽位于面板 16px 内边距之内，不接触窗口圆角。今日进度、金币、宠物和技术状态不占用主界面。

展开后的子代办支持单次添加、标题原位修改、未完成子项软删除、完成与撤销，以及同一父项内部的拖动和 `Alt+↑/↓` 重排。完成子项降低对比度并加删除线，但留在原顺序；第一版不允许跨父项移动、父子转换或继续嵌套。

完成流程：

```text
勾选独立代办
→ 写入该代办的完成事件，金币账本在后端同事务记账 `+1`

勾选父代办
→ 自动为仍未完成的有效子项写入完成事件，子项不产生金币
→ 已完成子项不重复写，已软删除子项不参与
→ 最后写入父项完成事件、金币 `+1` 和唯一主命令回执
→ 全部写入同事务成功或回滚，整组随后离开当前列表
→ 新的顶层第一项进入胶囊，已完成页父行以 `↶` 提供撤销

勾选子代办
→ 写入子项完成事件，不产生金币
→ 子项留在父项原位置，父项进度和胶囊当前动作刷新
→ 最后一条子项完成后也不会自动完成父项
```

撤销完成时同时回退奖励，不能重复发币。

调整顶层顺序时拖动父项右侧手柄，整个父子组作为一个单位移动；子代办使用自己的低存在感手柄，只能在所属父项内部排序。纯键盘操作都使用 `Alt+↑/↓`，但顶层和子项使用彼此独立的完整顺序。`reorder_tasks`、`reorder_subtasks` 与其他写操作共用可靠操作箱，结果未知时不会重复改序。

待办较多时按 `Ctrl+F`，或从更多菜单选择“搜索待办”；快速记录区临时替换为共享搜索栏，同时匹配父标题和子标题，并用克制的暖色标出命中片段。命中子项时必须保留父行作为上下文并临时展开命中子项，进度仍显示整组真实值；命中父项时可查看完整子列表。已完成页使用同一入口和位置，菜单文案切换为“搜索已完成”，同时匹配有效父完成与子完成的标题快照。空查询保留搜索态并显示完整列表；无命中时分别显示“没有找到相关待办”或“没有找到相关完成记录”。`Esc` 或“取消”退出搜索，中文输入法组合期间不会用半成品刷新列表，也不会把候选框的 `Esc` 误当成退出。

搜索只改变当前页面的只读投影，不改变账本快照、父子关系和原顺序。筛选结果仍可完成、修改、删除或撤销，但搜索期间不能新增子代办，顶层与子项排序柄都隐藏，拖动和 `Alt+↑/↓` 也由控制器明确拒绝，避免把局部结果提交成完整顺序。完成、删除或改名让最后一个命中项消失时，焦点回到搜索框；“当前待办”标记仍严格以真实 `currentTask.id` 为准，不把筛选后的第一项冒充当前项。

删除流程：

```text
点击未完成子代办行的“×”
→ 子项软删除并从进度分母排除，不产生金币

点击没有子代办的父项“×”
→ 父项直接软删除并从当前队列移除

点击含子代办的父项“×”
→ 显示“删除整个任务组”确认层
→ 确认后只写一条父项软删除，整组从当前列表隐藏
→ 子项投影和已有完成记录继续保留
```

除含子代办父项的整组删除外，删除入口保持轻量，不弹出额外确认、金币或完成反馈，也不伪装成完成。已完成子代办必须先撤销完成后才能删除；前端暂未提供删除撤销入口。

标题修改流程：

```text
双击未完成待办标题，或聚焦标题后按 Enter / F2
→ 原位置切换为单行标题输入框并聚焦，同时显示可选日期控件
→ Enter 或输入框失焦时提交有效变化
→ 后端追加 title_updated 事件并刷新真实快照
→ 任务仍留在原顺序，不改变状态和金币
```

编辑时按 `Esc` 会恢复原标题并结束编辑，不提交命令，也不会继续触发窗口收起。空白标题不会提交：按 `Enter` 时保留编辑框并标记无效，失焦时取消；与原题相同的标题直接取消。输入法组合输入期间的 `Enter` 不得提前保存。标题修改仅面向当前统一列表中的立即待办，完成记录和其他非立即状态不提供入口。

子代办沿用相同的双击、`Enter` / `F2`、失焦保存、`Esc` 取消和输入法保护，但只显示单行标题输入框，不显示期限控件；已完成子代办必须先撤销完成才能修改。父项原位修改态继续保留父行固定 `＋`，不在编辑区域复制第二个“添加子代办”入口。

期限展示与编辑流程：

```text
新任务默认 deadlineOn=null
→ 常态列表和胶囊不显示日期
→ 双击标题、按 Enter/F2，或点击任务行已有期限标签进入原位编辑
→ 点击胶囊标题行末尾的独立 edit-current-deadline 按钮时，一次完成展开面板并聚焦当前任务日期输入
→ 编辑期间隐藏原期限标签
→ 日期控件只接受 YYYY-MM-DD；清空表示 null
→ 有效变化通过 update_task_deadline 提交
→ 后端追加 deadline_updated，并刷新真实快照
→ 日期选择或清除成功后，焦点回到任务标题
```

已有期限按当前本地日期紧凑显示：“今天”“明天”、同年其他日期 `M/D`、跨年日期 `YYYY/M/D`、过去日期“逾期 N 天”。这些标签只做展示和编辑入口；设置、改期、清除、逾期或午夜刷新都不会自动排序、隐藏任务、提醒、通知、同步日历、发放或扣除金币。逾期和午夜刷新由前端从 `deadlineOn` 派生，不追加领域事件。期限与“稍后再做”的 `deferUntil` 完全独立，不决定任务是否留在立即执行队列。仅当前可见的立即待办可修改期限；完成、撤销完成和删除只改变原有状态语义，保留任务已有期限字段。

一级子代办不设置、继承或推导独立期限。胶囊显示的日期始终属于父项；子项期限、提醒和日历联动均为后续候选，不进入当前工作区实现。

### 5.2 快速记录

当前入口是展开面板中的单行输入：输入一句话，按“记下”或回车后进入队尾。当前没有“全局新增待办”快捷键；`Ctrl+F` 已专用于当前待办页或已完成页搜索。

标题是唯一必填项。快速记录中的空内容、重复回车和保存失败必须有明确但不打断的处理。行内标题编辑激活时，`Esc` 优先取消编辑；只有没有编辑态时，界面壳才继续处理收起临时页面或窗口的后续 Esc 语义。

顶部快速记录永远创建顶层代办，不解析缩进、列表符号或多行粘贴。子代办统一从父行固定 `＋` 进入单次添加态，输入框位于子列表末尾：非空标题按 `Enter` 或输入框失焦时创建，成功后退出添加态并保持列表展开，创建失败则保留原输入。空白或纯空格失焦以及 `Esc` 都直接取消；没有旧子项时同时收起空列表，已有子项时保持展开。子列表底部不再重复显示新增入口，子标题修改仍按 `Enter` 或失焦保存。

### 5.3 后续：暂时做不了

以下状态语义已经设计，但当前桌面版本尚未开放命令和界面：

| 操作 | 第一版输入 | 处理结果 |
|---|---|---|
| 稍后再做 | 明天、下周、自定义时间 | 到期前跳过，不强制替换当前任务 |
| 被卡住 | 简短原因 | 进入阻塞区，可恢复处理 |
| 不再做 | 调整原因 | 进入已放弃，可重新打开 |

三种处理均不发金币、不扣金币，也不影响宠物成长；确认后下一条可执行任务成为当前任务。

### 5.4 暂停：宠物小屋

宠物阶段恢复后，第一版只包含金币、少量食物、成长值和成长阶段：

```text
完成任务获得金币
→ 花费金币喂食
→ 成长值增加
→ 达到阈值后成长
```

宠物不会死亡、掉级、饿坏或因长时间未使用责备用户。金币不足时只给温和提示。

### 5.5 已完成、复制本周完成与后续周报

当前已实现独立“已完成”页。独立待办保持普通单行；含子代办的记录按父项分组并默认折叠，摘要显示父标题、真实进度、时间和展开入口。展开后显示有效子完成明细；父项未完成时可撤销子项，父项已完成时必须先撤销父完成，父项被删除后只保留只读历史。进入该页时，顶部应用标题直接替换为“‹ 已完成”，不再增加第二层页面标题；只有记录列表内部滚动，并与待办页复用同一条内缩细滚动条规则。按 `Ctrl+F` 或更多菜单会在标题栏下方打开共享搜索栏；命中子标题时只临时展开对应组，退出搜索后恢复原折叠状态。待办页入口右侧总数仍表示全部有效完成事实，不被筛选结果改写。它读取本地事件账本，不依赖网络。

“复制本周完成”属于低频周回顾操作，当前只在已完成页的右上角更多菜单中出现，不占用标题栏。`0.1.2` 起按父项去重：父项在本周最终完成时，父项只输出一次，属于该父项且仍然有效的子完成可以早于本周，并作为缩进明细；父项尚未完成时，只有本周内完成的普通子项才以“父项 / 子项”进入结果。已撤销完成不会输出。写入成功提示中的 `N` 表示生成的 Markdown 顶层编号数，不是原始子完成事件数；本周没有有效结果时不会覆盖现有剪贴板内容。

这只是完整非 AI 周报的第一个最小切片。后续周报再按“本周完成、进行中、阻塞事项、计划调整”分组，保留时间和原因，并继续支持：

- 今天、上周和自定义日期。
- 选择是否纳入单项。
- 查看单任务事件时间线。
- 复制纯文本或包含更多分组的 Markdown。
- 不依赖 AI 和网络生成基础周报。

### 5.6 本地数据边界

`0.1.1` 使用数据库 v4；`0.1.2` 把本地结构升级为 v5，并增加父子关系、同级位置、完整子项投影和有效完成事实投影。v4 数据迁移时先通过 `VACUUM INTO` 生成并验证一致性备份。这个备份只服务于数据库升级，不等于通用自动备份；手动导出、定期备份和未来更多版本迁移仍是后续能力。

## 6. 极简视觉系统

### 色彩

- 页面和系统背景：中性浅灰。
- 任务面板：白色或轻微系统材质。
- 文字：深灰，不使用纯黑大面积压迫。
- 交互强调：低饱和系统蓝或绿色。
- 当前暖色品牌点：只做小面积提示；未来宠物与金币启用后也不能扩散成大面积装饰。
- 错误与逾期：仅使用小面积提示，不使用大面积红色。

### 字体与密度

- 中文正文优先使用系统清晰字体。
- 数字使用清晰、稳定的字形，便于阅读数量和时间。
- 任务标题字号明显高于元信息，但不使用展示型大标题。
- 桌面层保持紧凑，完整主窗口保持舒适行高。

### 圆角与阴影

- 圆角克制，避免所有组件都变成胶囊。
- 阴影只表达悬浮层级，不能形成厚重卡片堆叠。
- 透明是增强效果，关闭系统透明后仍需清楚可用。

### 动效

- 当前完成反馈保持短暂且不打断操作。
- 后续喂食反馈建议为 0.6～1.2 秒，待机宠物只做小幅呼吸或眨眼。
- 支持减少动态效果。
- 默认无声音、无全屏撒花、无必须关闭的奖励弹窗。

## 8. P1 规则与开发验证

以下内容需要有明确规则，但不要求本轮全部做成高保真页面：

- 超长标题、任务很多和空队列。
- 无子项父行只保留固定 `＋`，不显示 `0/0`；父子长列表仍只使用一个滚动区。
- 一级子代办的固定新增入口、单次添加态、空输入取消后的折叠规则、父项自动完成剩余子项、父子独立排序域、搜索上下文、分组历史、胶囊动作和整组删除确认。
- 标题修改的双击、键盘进入、失焦保存、取消、输入法组合态和重复提交保护。
- 期限控件的设置、改期、清除、同值拒绝，以及今天/明天/同年/跨年/逾期和午夜刷新展示。
- 今天、上周、自定义区间和跨月/跨年的完整周报。
- 多行迁移预览、重复标题和迁移结果。
- 备份恢复预览、成功和失败。
- 浅色、深色和高对比主题。
- 100%～200% 缩放、多显示器、任务栏不同方向；当前仅单屏 125% 实机通过。
- 拔掉副屏后的窗口回收；当前纯函数边界测试通过，实机待补。
- 全屏自动隐藏；单屏 125% 下，胶囊态与展开态退出后的冷启动位置归一化均已通过；整窗鼠标穿透已从产品与代码中移除。
- 减少动态效果和更完整的纯键盘操作；当前重排已支持 `Alt+↑/↓`。

## 9. 第一版发布阻断项

以下四项任一失败，极简原生版本都不适合作为长期桌面常驻工具：

1. 悬浮窗抢走其他应用的键盘焦点。
2. 收起态过大、遮挡系统托盘或常用按钮。
3. 快速记录不能在约三秒内完成。
4. 多显示器、缩放或重启后窗口位置无法可靠恢复。

## 10. 后续皮肤适配

极简原生主流程和状态稳定后，深色透明与像素桌宠按照同一状态矩阵适配。皮肤适配不能复制业务逻辑，也不能改变“完成”“隐藏”“阻塞”等操作语义。
